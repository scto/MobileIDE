Instruction for Templates Submodule Generation in GitHub Repository: ://github.com
Target Stack: Android (minSdk 26, targetSdk 35), Gradle 8.11.2 (Kotlin DSL), Java 17
Scope: Stateless Project Generator with Koin Dependency Injection, Clean MVI Architecture, and Version Catalogs.
Path Constraints: Output targets the shared system '$PROJECTS' directory.

Generate the following architecture under /templates-api/ and /templates-impl/:

1. INTERFACES & CONFIGURATION VIA KOTLIN FLOWS (/templates-api/)
   - Create `ProjectTemplateEngine` interface returning a `Flow<GenerationStep>`.
   - Define data class `TemplateConfig`: projectName, packageName, minSdk=26, targetSdk=35, compileSdk=35, versionCode, versionName, useKotlinDSL=true, language=KOTLIN, architecturePattern=MVI, templateType (BOTTOM_NAV/NAV_DRAWER/EMPTY).
   - Define sealed class `GenerationStep`: Processing(val message: String), Completed(val message: String), Failed(val error: String).

2. ASSETS STRUCTURE AND STRUCTURED INJECTION (/templates-impl/src/main/assets/project-templates/)
   
   A. DEPENDENCY MANAGEMENT & BUILD SCRIPTS (/common/)
      - Create `/common/gradle/libs.versions.toml.txt`: 
        Configure Android Gradle Plugin 8.7+, Gradle 8.11.2, Material 3, Compose BOM, Lifecycle-Runtime-Ktx. 
        Add strict version definitions and library bundles for Koin core, Koin Android, and Koin Compose (`io.insert-koin:koin-androidx-compose`).
      - Create `/common/app-build.gradle.kts.txt`: 
        Standard app-level script applying application and kotlin-android plugins via aliases. Set compiler toolchains strictly to Java 17. Include standard compose compiler configurations.
      
   B. KOIN DEPENDENCY INJECTION ARCHITECTURE (/di/ & /)
      - Create `/common/di/AppModule.kt.txt`: 
        Define a global Kotlin Koin module layout initializing view models dynamically using Koin's structural reflection DSL (`viewModelOf(::MainViewModel)`).
      - Create `/common/MainApplication.kt.txt`: 
        Extend Android's base `Application` class. Override `onCreate()` to invoke `startKoin {}`, registering the `androidContext`, loading the `androidLogger`, and linking the pre-defined `appModule`.

   C. CLEAN MODEL-VIEW-INTENT (MVI) FRAMEWORK CORE (/architecture/mvi/)
      - Create `/architecture/mvi/MviArchitecture.kt.txt`: 
        Define immutable foundational boilerplate core interfaces: `UiState`, `UiIntent`, and `UiEffect`.
      - Create `/architecture/mvi/BaseViewModel.kt.txt`: 
        Implement a production-ready, thread-safe abstract class leveraging Kotlin Coroutines. Use `MutableStateFlow` to host an absolute immutable State matrix and `MutableSharedFlow` to pipeline unidirectional UI Intents safely across lifecycle bounds.

   D. MATERIAL DESIGN 3 COMPOSABLE LIFECYCLE (UI LAYOUTS)
      - Create `/ui-styles/bottom-navigation/MainActivity.kt.txt`: 
        Configure the main launch pad layout using Material 3 `Scaffold` and a responsive `NavigationBar`. 
        Inject the modern state lifecycle pattern using Koin's `koinViewModel()` inside the top-level Composable compositing tree, decoupling direct instance state creation from the screen UI tree.

3. GENERATOR PARSING ENGINE LIBRARIES (/templates-impl/src/main/java/.../)
   - Implement `ProjectTemplateEngineImpl(private val context: Context) : ProjectTemplateEngine`.
   - Architect an asynchronous coroutine execution structure:
     * Convert dot-separated package strings (`com.example.app`) safely into system folder mapping directories (`src/main/java/com/example/app/`). Create folders inside the destination path.
     * Open static asset data structures safely via Android's `AssetManager`. Run a memory-efficient stream loop parsing text, exchanging configuration variables (`${packageName}`, `${projectName}`, `${minSdk}`).
     * Automatically output the core MVI architecture templates, wire up the Koin DI container paths, inject the base `MainApplication` configurations into the generated `AndroidManifest.xml`, and export everything directly into the designated `$PROJECTS` folder structure.

Register `:templates-api` and `:templates-impl` inside the root build graph files. Ensure complete compilation decoupling via explicit implementation contracts. Generate all files completely.
